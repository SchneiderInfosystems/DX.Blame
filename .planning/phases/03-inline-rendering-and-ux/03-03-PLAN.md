    Dialog behavior:
    - On show: populate all controls from BlameSettings properties
    - On OK: write all control values back to BlameSettings, call BlameSettings.Save, call InvalidateAllEditors (from DX.Blame.Renderer), ModalResult := mrOk
    - On Cancel: ModalResult := mrCancel, no changes
    - Form size: approximately 400x480, Position := poScreenCenter, BorderStyle := bsDialog

    Write the .dfm file manually (not via IDE designer). Use Delphi DFM text format with proper structure. Sonderzeichen in DFM must use Delphi #NNNN codepoint syntax. Keep labels and captions in English.

    Class method for easy invocation:
    ```pascal
    class procedure TFormDXBlameSettings.ShowSettings;
    ```
    This creates the form, shows modal, frees on close.

    **DX.Blame.Navigation.pas:**
    Create the parent revision navigation feature:

    ```pascal
    procedure NavigateToParentRevision(const AFileName: string;
      const ACommitHash: string; const ARepoRoot: string);
    ```

    Implementation steps:
    1. Resolve parent hash: Execute `git rev-parse <ACommitHash>^` via TGitProcess. If this fails (no parent -- root commit), show IDE message and exit.
    2. Get relative file path: compute path relative to ARepoRoot.
    3. Get file content at parent: Execute `git show <ParentHash>:<RelativePath>` via TGitProcess.
    4. Write to temp file: Path = `%TEMP%\DX.Blame\<BaseName>.<ShortHash>.pas` (use first 7 chars of parent hash). Use ForceDirectories. Write content as UTF-8 with BOM via TFile.WriteAllText.
    5. Open in IDE: Use IOTAActionServices.OpenFile(LTempFile).
    6. After opening, trigger blame on the temp file so annotations appear in the parent revision tab too.

    Additional helper:
    ```pascal
    function IsParentRevisionAvailable(const ACommitHash: string): Boolean;
    ```
    Returns False if ACommitHash is empty or equals cUncommittedHash.

    Context menu integration helper:
    ```pascal
    procedure AttachContextMenu;
    procedure DetachContextMenu;
    ```
    Find the editor popup menu via FindComponent pattern (GExperts approach):
    - Get the active edit window form
    - FindComponent('EditorLocalMenu') to get the TPopupMenu
    - Add a TMenuItem "Previous Revision" with OnClick handler
    - OnClick: determine current file and line, get blame data for that line, call NavigateToParentRevision
    - The menu item should be disabled/hidden when the current line is uncommitted (IsUncommitted = True)

    For determining the current line and file in the context menu handler:
    - Use IOTAEditorServices.TopView.CursorPos.Line for line number
    - Use IOTAEditorServices.TopView.Buffer.FileName for file path

    All .pas files: UTF-8 with BOM, CRLF line endings, standard MIT header.
    .dfm file: CRLF line endings, standard Delphi text DFM format.
